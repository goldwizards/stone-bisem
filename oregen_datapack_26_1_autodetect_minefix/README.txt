# 26.1 업그레이드형 조약돌 생성기 데이터팩 (자동 탐지 + 채굴 후 1회 판정)

## 지원 버전
- Minecraft Java Edition 26.1
- pack format: data pack 101.1

## 핵심 동작 흐름 (중요)
1. 플레이어 주변에서 생성기 출력 칸 후보를 자동 탐지
2. 물/용암 인접 구조가 맞으면 marker로 출력 칸 등록
3. 등록 시점에 이미 있던 stone/cobblestone은 그대로 유지
4. 출력 칸이 비어야(air/cave_air/water/lava) `ready=1`
5. 이후 새 stone/cobblestone이 재생성될 때만 1회 판정
6. 판정 후 즉시 `ready=0`으로 내려 재판정 방지

즉, **탐지 직후 즉시 변환되지 않고**
**채굴 → 재생성된 새 블럭에만 1회 변환** 됩니다.

## 업그레이드 시스템
- 업그레이드 블럭은 소모되지 않음
- 부수면 즉시 다운그레이드
- 누적 조건(상위 레벨은 하위 포함)
- 생성기 주변 범위 제한 없음
- 월드 전체 공용 업그레이드 풀(모든 생성기가 공유)

### 레벨 조건
- 1레벨: 기본
- 2레벨: 석탄블럭 7개
- 3레벨: 석탄 7 + 철 7
- 4레벨: 석탄 7 + 철 7 + 금 7
- 5레벨: 석탄 7 + 철 7 + 금 7 + 다이아 7
- 6레벨: 석탄 7 + 철 7 + 금 7 + 다이아 7 + 에메랄드 7

### 해금 구조
- 1레벨: 돌, 석탄
- 2레벨: 철 해금
- 3레벨: 금 해금
- 4레벨: 다이아 해금
- 5레벨: 에메랄드 해금
- 6레벨: 고대 잔해 해금

## 설치 방법
1. 데이터팩 폴더(또는 zip)를 월드의 `datapacks` 폴더에 넣습니다.
2. 월드에서 `/reload`를 실행합니다.
3. `[광물 생성기] 데이터팩 로드 완료 ...` 메시지가 뜨면 정상입니다.

## 사용 방법
- 자동 탐지는 약 10틱마다 플레이어 주변에서 수행됩니다.
- 출력 칸 주변에 water/lava가 맞게 인접하면 자동 등록됩니다.
- 수동 등록도 가능:
  - `/function oregen:player/register`
  - `/function oregen:player/unregister`
  - `/function oregen:player/info`

## 함수 구조
- `oregen:load`
- `oregen:tick`
- `oregen:detect/*`
- `oregen:generator/*`
- `oregen:level/*`
- `oregen:roll/*`
- `oregen:upgrade/*` (업그레이드 블럭 증감 이벤트)

## 참고 사항
- 업그레이드 블럭 카운트는 블럭 설치/파괴 이벤트(advancement)로 관리됩니다.
- 따라서 데이터팩 적용 **이전에** 이미 놓여 있던 업그레이드 블럭은 자동 집계되지 않을 수 있으며,
  그 경우 한 번 다시 설치/파괴해 카운트를 동기화하세요.
- waterlogged 블럭의 모든 변형을 완전 판별하는 대신, 생성기 핵심인 water/lava 인접 구조를 우선 탐지합니다.
